# Mission Control Plan 功能说明（面向小白）

## 背景

为了让 Mission Control 更适合非技术用户（小白）使用，需要在任务正式执行前，把“怎么做”以可视化清单的方式展示出来，并且允许用户在执行过程中参与控制。

## 核心目标

在任务创建后、执行前，增加一个 **Plan（执行计划）** 区块，成为任务 UI 的一部分，行为类似 Claude Code 的 plan：

- Plan 由多条子任务组成；
- 每条子任务前有 checkbox；
- 正在执行中的步骤有明显状态提示；
- 完成步骤显示 ✅；
- 失败步骤显示 ❌；
- 用户可以在运行过程中与 plan 交互（例如终止某一项任务）。

## 交互要求

### 1) 执行前

- 用户可看到完整计划清单；
- 每个计划项都有独立状态（待开始 / 进行中 / 已完成 / 失败 / 已终止）；
- UI 应避免技术术语，文案尽量直白。

### 2) 执行中

- 当前执行项要高亮；
- 执行状态需实时更新；
- 用户可在 plan 项级别触发交互（如终止某一项）。

### 3) 执行后

- 保留完整计划轨迹，方便复盘；
- 失败项要可识别并可追踪原因（至少保留失败标记）。

## 状态建议（用于前后端对齐）

建议 plan item 采用以下状态枚举：

- `pending`：待开始
- `running`：进行中
- `done`：已完成（✅）
- `failed`：失败（❌）
- `aborted`：用户终止

## UI 最小呈现建议

每个 plan item 最少包含：

- 状态图标（checkbox / ✅ / ❌ / 停止标记）
- 标题（一句话说明动作）
- 可选操作按钮（终止）

> 本文档根据 MC-1 任务描述整理，用于明确 Plan 功能的产品与交互要求。
